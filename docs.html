Project Overview

The AutoToy project is a DIY self-driving toy car platform that demonstrates fundamental principles of autonomous vehicles. Using affordable off-the-shelf components, this project implements computer vision, sensor fusion, and machine learning to create a fully autonomous navigation system.

The goal of this project is to create an accessible platform for learning about robotics, computer vision, and autonomous navigation algorithms in a hands-on way. The modular design allows for easy customization and expansion of capabilities.

Hardware Components

The AutoToy platform is built using the following components:

Core Components

  • Computing Unit: Raspberry Pi 4 (4GB RAM)
  • Motor Controller: L298N Dual H-Bridge
  • Power Supply: 7.4V LiPo Battery
  • Chassis: 4WD Robot Car Chassis Kit

Sensors

  • Vision: Raspberry Pi Camera Module v2
  • Proximity: HC-SR04 Ultrasonic Sensors (×2)
  • Line Following: TCRT5000 IR Sensors (×3)
  • IMU: MPU6050 (Accelerometer + Gyroscope)

Optional Components

  • Display: 0.96" OLED SSD1306
  • Communication: ESP8266 WiFi Module
  • GPS: NEO-6M GPS Module

Software Architecture

The AutoToy software stack consists of several modules organized in a layered architecture:

Perception Layer

Handles all sensor input processing:

  • Computer vision pipeline (OpenCV)
  • Sensor data fusion
  • Environmental mapping

Decision Layer

Processes sensor data to make navigation decisions:

  • Path planning algorithms
  • Obstacle avoidance
  • Machine learning models (TensorFlow Lite)

Control Layer

Translates decisions into motor commands:

  • Motor PWM control
  • PID controllers for precise movement
  • Safety override systems
